Project Plan: Don't Wreck My House

High-level Requirements

  • View Reservations: Display all reservations for a host.
  • Add a Reservation: Create a new reservation for a guest.
  • Edit a Reservation: Modify an existing reservation.
  • Cancel a Reservation: Remove a reservation from the system.

Data Model

Reservation

  • ID: Unique identifier (Integer)
  • Start Date: Start of the reservation (LocalDate)
  • End Date: End of the reservation (LocalDate)
  • Guest: Guest object
  • Host: Host object
  • Total: Cost (BigDecimal)

Host

  • ID: Unique identifier (UUID or String)
  • Last Name: Last name (String)
  • Email: Email address (String)
  • Phone: Phone number (String)
  • Address: Address (String)
  • City: City (String)
  • State: State (String)
  • Postal Code: Postal code (String)
  • Standard Rate: Regular rate (BigDecimal)
  • Weekend Rate: Weekend rate (BigDecimal)

Guest

  • ID: Unique identifier (Integer)
  • First Name: First name (String)
  • Last Name: Last name (String)
  • Email: Email address (String)
  • Phone: Phone number (String)
  • State: State (String)

Validation Rules

  • Dates: Start date before end date, both in future.
  • Guest and Host: Must be valid in the system.
  • No Overlaps: No overlapping reservations for the same host.
  • Email: Validate proper format.
  • ID: Must be a positive integer.

Technical Requirements

  • Architecture: MVC structure.
  • Storage: CSV files for reservations.
  • Exceptions: Custom exceptions for errors.
  • DI: Dependency Injection for services.
  • Testing: Unit & Integration tests with JUnit 5.
